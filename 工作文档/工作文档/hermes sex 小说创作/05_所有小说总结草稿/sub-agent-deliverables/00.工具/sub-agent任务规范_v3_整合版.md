## §1 子代理执行流程（强制顺序，不可跳过）

```
步骤1：ls 定位源目录 → 确认文件存在
  ↓
步骤2：分批读取源小说（每批 ≤ 3-5章 或 单部作品）
  ↓
步骤3：读完一批立即 Write 写入草稿文件（不积压）
  ↓
步骤4：重复步骤2-3直到全部读完
  ↓
步骤5：末尾追加 5 行完成报告
  ↓
步骤6：向主代理回报简短完成摘要
```

### ⚠ 核心强制规则：实时写入，禁止积压

**为什么要这样做**：
- v2.3实战证明：单SA读完所有文件再写会导致上下文爆炸、压缩、截断、丢失早期细节
- 读完3-5章或1部作品立即写入，释放上下文空间

**分批规则**：
| 场景 | 分批粒度 | 写入时机 |
|---|---|---|
| 单部独立作品 | 整部 = 1批 | 读完立即写1份 |
| 多部聚合包 | 1部 = 1批 | 每读完1部立即写，N部=N份内容追加到同一文件 |
| 超长作品(>500KB) | 每3-5章 = 1批 | 每读完一批立即追加写入 |
| 多文件系列子段 | 1个文件 = 1批 | 每读完1个文件立即追加 |

**绝对禁止**：
- ❌ 读完所有章节/所有作品再统一写（会爆上下文）
- ❌ 每读完1章就写1份（过度碎片化）
- ❌ 边读边写（注意力分散丢失细节）
- ❌ 读取`深度阅读笔记/`目录下任何文件（笔记是输出不是输入）
- ❌ 读取其他SA的源小说或草稿
- ❌ 自创R1/R2/R3之外的雷点标签

## §2 9段标准输出结构（每部作品写一套）

> 多部聚合包：N部作品 = N套完整9段，按顺序追加到同一个草稿文件。
> 段号固定，顺序不可变，缺一不可。

| 段号 | 段名 | 硬性要求 | 核心内容 |
|---:|---|:---:|---|
| **§1** | **作品定位** | — | 类型/流派/情绪基调/目标读者画像/核心卖点 |
| **§2** | **关键章节** | 表格 | 列：章节号 / 标题 / 剧情浓度(1-5) / 核心看点；选3-10个关键章节 |
| **§3** | **完整套路** | 5-12段 | 核心叙事模式 + 反转节点 + 情绪曲线 + 节奏推进公式 |
| **§4** | **完整情节** | — | 梳理完整剧情脉络，分阶段概括 |
| **§5** | **重复模式/模板化桥段** | — | 归纳作品中重复出现的叙事范式、固定桥段 |
| **§6** | **关键R18情节元素总结** | **5-15条，每条总结原文≥300字（硬要求）** | 摘录/改写/总结原文关键段落，保留动作/对白/心理/核心细节（姿势/部位/节奏/主导权） |
| **§7** | **内容标签判定** | **只用R1/R2/R3三个标签，判定命中/不命中** | 见§3雷点判定细则 |
| **§8** | **写作技法** | 3-8项，自由发挥 | 技法名称 + 使用章节 + 效果分析 + 可复用建议 |
| **§9** | **SKILL升级建议** | **≥2条（硬要求）** | 具体可操作建议，对应输出模板/套路库可补充的内容 |

## §4 §6关键R18情节元素总结硬要求

> ⚠ 术语已统一：**"总结原文"**（不是"整理原文"）

- **条数**：5-15条（根据作品R18浓度决定，不得少于5条）
- **每条字数**：≥300字（硬性要求，不足需补充）
- **内容要求**：总结原文关键段落，必须保留：
  - 核心动作（姿势/部位/节奏）
  - 角色对白（关键对话原文/改写）
  - 心理描写（主角/攻方/受方心理变化）
  - 主导权关系（谁主导、如何推进）
  - 关键细节（感官描写、环境、道具）
- **R18元素可以如实总结**，不需要回避
- **严禁**：
  - G元素（去势/刀割/断肢/极端血腥）——R2雷
  - 父子之间任何R18描写——R3雷
  - 女性角色作为R18主体——R1雷（默认male化处理）

## §6 完成报告格式（草稿末尾追加5行）

```markdown

- **处理小说**：{N}部（{作品列表}）
- **实际读取章节**：{X} / {总章节数}章

- **草稿文件大小**：kb
- **异常情况**：{章节数偏差/路径问题/其他，无则填"无"}
```

向主代理回报时只回复一行摘要：
```
[SA-03-{ID}] 完成：N部 / 写入§X.X / 总大小~XXXKB / 标签N处 / 用时X分钟
```
