## Chinese-WebNovel-Skill-2 完整分析总结

### 一、整体架构设计

这是一套面向中文网文写作的模块化 Skill 系统，采用「主 Skill 路由 + 专项模块下沉 + 本地语料检索」的三层架构设计，而非单一超长 Prompt。

**核心设计理念：**
- 主 SKILL.md 仅保留全局原则、流程约束和模块路由
- 每个高频写作问题拆分为独立专项模块
- 提供 166 篇本地小说语料（664 条结构化摘录）作为模仿基础
- 模型先诊断问题层级，再调用对应模块和素材

**目录结构四层：**
1. **SKILL.md**：路由层，全局原则、流程约束、模块调用总则
2. **references/modules/**：10 个专项模块，每个模块独立包含教程、运行规则、正反例
3. **data/ + analysis/**：本地小说语料、结构摘录、检索索引
4. **scripts/**：检索脚本，支持「先匹配素材 → 再构思 → 再写作」工作流

---

### 二、模块链路与职责分工

系统将网文写作分为三段链路，共 10 个专项模块：

#### 第一链：前置规划链（上游决策层）

| 模块 | 核心作用 | 阶段定位 | 解决问题 |
|------|---------|---------|---------|
| **concept_planning** | 把一句简介压成题材、消费点、hook、premise、故事引擎、长度判断和第一卷方向 | 最上游 | 点子值不值得写、卖点亮不亮、引擎能不能跑 |
| **opening** | 把卖点、异常局面、主角亮相落到前 300-3000 字开头成交区 | 开篇执行 | 开头没抓手、卖点露头慢、黄金三章起得慢 |
| **volume_outline** | 把故事引擎展开成黄金三章、分卷设计、前 10-20 章章纲 | 中长线结构 | 章纲散、中段发散、卷末没兑现点 |

#### 第二链：正文执行链（中游生产层）

| 模块 | 核心作用 | 优先级 | 解决问题 |
|------|---------|-------|---------|
| **plot_logic** | 修动机→触发→决策→后果→兑现因果链 | 最高（底层） | 因果断裂、动机不足、触发不成立、后果不落地 |
| **character_consistency** | 修目标/情绪/关系/身体/声音五类人物连续性 | 最高（底层） | 人设崩、情绪跳变、关系断片、伤势遗忘、声音漂移 |
| **transition** | 处理时间跳切、空间切换、情绪承接、视角切换、章末接下章 | 中 | 转场生硬、空间突兀、情绪断裂、视角插播 |
| **dialogue** | 处理关系压力、人物声音、信息嵌入、对白刀口 | 中 | 台词空、像说明书、人物一个腔调、有情绪没刀口 |
| **chapter_ending** | 处理章末拉力、余韵、回钩、下章承接 | 中 | 章末太平、收在总结讲道理、没追更欲 |
| **anti_ai_voice** | 清理空泛总结、套话氛围、说明书对白、统一腔调 | 最低（表层） | AI味重、模板化、句子太平均、形容词堆砌 |

#### 第三链：完稿收口链（下游质检层）

| 模块 | 核心作用 | 强制执行 |
|------|---------|---------|
| **consistency_review** | 统一复查六种一致性 | 每章写完必过 |

**模块调用优先级原则：**
- 先修底层结构问题（plot_logic / character_consistency）
- 再修场景执行问题（transition / dialogue / chapter_ending）
- 最后修表层文风问题（anti_ai_voice）
- 绝对不能把结构问题误修成文风问题

### 三、核心写作方法论

#### 1. 前置规划方法论（concept_planning）

**核心定义：** 前置规划不是写大设定，而是把简介压成能连载、能拆章、能继续写的最小骨架。

**必须回答的六个问题：**
1. **题材和消费点**：读者主要来吃什么？爽点/甜点/刀点在哪？
2. **卖点和异常局面**：谁？在什么异常局面？因为什么必须做什么？最抓人点是什么？
3. **主角眼前麻烦**：最紧迫问题是什么？不动会失去什么？前三章撞什么麻烦？
4. **故事引擎**：接下来10-20章靠什么持续推进？（升级/调查/身份反打/关系拉扯/倒计时）
5. **目标长度**：适合短篇/中篇/长篇/连载？有没有足够阶段性变化？
6. **最小产物**：一句话hook、一段premise、主角当前目标、主要阻力源、故事引擎说明、第一卷最小目标

**最小骨架卡（动笔前必填）：**
- 这是一本什么题材的网文？
- 读者主要来吃什么？
- 主角眼前最紧迫的问题是什么？
- 接下来10-20章靠什么持续推进？
- 更适合什么长度？
- 第一卷最小目标和卷末变化是什么？

#### 2. 开头成交方法论（opening）

**核心定义：** 网文开头不是铺垫区，而是成交区。必须尽快让读者知道三件事：主角是谁？现在出了什么异常？这本书最值得追的东西是什么？

**四种开头成交方式：**
- **危机压身型**：一上来就让主角处在要付代价的局里，卖点和危机绑定
- **身份反差型**：让主角身份本身就是冲突，反差立刻落成局面
- **信息外挂型**：用弹幕/系统/帖子/爆料/录音等外来信息改轨主角判断
- **关系炸点型**：直接从关系里最危险的刀口切入，一句话改写关系位置

**硬性指标：**
- 前 300-500 字必须给抓手
- 前 1500-3000 字必须给卖点
- 前 6000 字必须让读者知道：主角是谁、遇到什么、卖点是什么、接下来想看什么
- 黄金三章递进：第一章立异常 → 第二章扩卖点 → 第三章立方向和追更预期

**开头最小计划：**
- 第一刀切在哪？
- 主角最具体麻烦是什么？
- 最早抛出的卖点是什么？
- 第一章结束局面怎么变？
- 第三章前让读者形成什么追更问题？

#### 3. 分卷章纲方法论（volume_outline）

**核心定义：** 分卷章纲不是备忘录，而是故事引擎的施工图。

**三个层级：**
- **黄金三章**：负责成交（主角亮相、卖点亮相、第一冲突亮相、阶段目标亮相、追更理由亮相）
- **分卷**：负责给故事分阶段目标（卷目标、卷核心冲突、卷高潮、卷末变化）
- **章纲**：负责把卷目标拆成可更新的章节推进（本章目标、本章冲突、本章新信息、本章章末拉力）

**一直盯着的四件事：** 故事引擎、阶段目标、兑现点、拉力

**分卷章纲最小计划：**
- 第一卷最核心要解决什么？
- 第一卷最强冲突是谁或什么？
- 卷末局面必须怎样变化？
- 前10章至少哪三次升级或反打不能缺？
- 哪几章最容易水，提前怎么防？

#### 4. 剧情逻辑方法论（plot_logic）

**核心定义：** 剧情逻辑不是"现实会不会发生"，而是"在这个故事已给出的前提下，这件事为什么会发生，而且为什么偏偏在这里发生"。

**五个核心环节：**
1. **动机**：人物想得到什么？不做会失去什么？为什么值得付成本？
2. **触发**：要有让人物"现在必须这么做"的那一下
3. **决策**：选择要和处境、性格、目标匹配
4. **后果**：选择之后局面必须变
5. **兑现**：前面种下的东西要以冲突/反转/代价/收益方式兑现（掉马/翻案/误会/打脸都属于兑现）

**最小因果链：**
- 现在人物最想得到什么？
- 什么东西逼他现在动？
- 他准备怎么做？
- 做了以后最直接的后果是什么？
- 这个后果会把下一章推向哪里？

#### 5. 人物一致性方法论（character_consistency）

**核心定义：** 人物一致性不是"人物永远不变"，而是"人物的变化要有来路，目标/情绪/关系/身体/说话方式能对得上"。人物可以变，但不能无故跳。

**五个核心维度：**
1. **目标一致**：这一章最想要什么，不能一会儿一个方向
2. **情绪一致**：情绪可以变化，但要有触发物和过渡
3. **关系一致**：关系温度变化需要中间机制
4. **身体一致**：伤/病/醉酒/疲惫/怀孕/失眠不能切场就忘
5. **声音一致**：说话习惯、出手方式、忍耐阈值相对稳定

**人物变化三要素：** 触发、过程、代价

**最小状态卡：**
- 他现在最想达成什么？
- 他现在最强烈的情绪是什么？
- 他和场上关键人物的关系温度是什么？
- 他身体上有什么不能忘？
- 他说话和行动最像什么风格？

#### 6. 转场方法论（transition）

**核心定义：** 转场不是一句过渡词，而是对时间、空间、情绪、信息和叙事镜头的重新组织。情节是车，转场是换挡。

**底层认知：**
- 转场本质是处理时间和空间压缩/折叠
- 好转场最重要的是"关联"（借时间/地点/动作/物件/画面/情绪/逻辑/共同事件搭桥）
- 转场其实是在切镜头（直接切/跟拍切/空镜切/借物切/回忆切/平行剪辑切）
- 好转场目标不是"说明白"，而是"接得住"（读者不迷路、人物不断片、情绪不掉线、下一场上来就有东西看）

**新手最易上手的5种方法：**
1. 排版转场（空一行）
2. 标志性短语转场（第二天/一个月后）
3. 对白预告下一场（明天见/晚上去找你）
4. 环境带入下一场（先给感官画面）
5. 情绪承接（上一场情绪带到下一场）

**进阶方法库：** 时间转场（直接标时/自然标时/时间压缩）、空间转场（直接转/过渡转/跳跃转）、细节/物件桥接、空镜转场、对话/话题引导、心理情绪转场、视角切换（必须有内在联系）、回忆梦境转场（从当下→心境→过去；被打断回来）、摄像机接力和平行剪辑、蒙太奇式转场

**网文适配6原则：**
1. 允许快，但不能乱
2. 新场景第一句就要有东西（事件/人物/冲突/结果/新信息）
3. 能省过程就省，不能省因果
4. 情感线最怕人物断片
5. 悬疑线可以故意突兀，但要有节奏感
6. 同一招别连用太多次

**转场前5问：**
- 上一场留下的未尽之力是什么？
- 下一场第一拍必须交代什么变化？
- 这次更适合借时间/空间/情绪/物件？
- 有没有共同触点连起两场戏？
- 该让读者明显感到切换，还是几乎察觉不到？

#### 7. 对白方法论（dialogue）

**核心定义：** 好对白不是聊天记录，也不是情绪宣泄。它要同时带出三样东西：关系位置、现实利益、情绪压迫。

**对白四大功能：**
1. **施压**：让场上压力可听见
2. **放信息**：信息嵌在对抗里出来，不是轮流讲解
3. **改关系**：一句话说完，双方位置发生偏移
4. **立人**：不同身份的人说法不一样

**四类高张力对白：**
- **地位压迫型**：借长辈/上位者/制度/家规/身份施压
- **情感绑架型**：用亲密关系/亏欠感/孩子/旧情/眼泪施压
- **利益摊牌型**：把钱/名声/婚姻/资源/退路直接摆上桌
- **冷静判决型**：不大吵，但一句话直接改局面

**对白五问：**
1. 说这句话的人现在想达到什么？
2. 这句话里有没有新东西？
3. 这句话说完，关系有没有偏移？
4. 这句话是不是只有情绪，没有刀口？
5. 换个人说，这句台词还能成立吗？

**对白最小计划：**
- 这一轮对话谁想赢什么？
- 他最想戳对方哪一刀？
- 对方最怕被戳什么？
- 这轮对话结束后，关系或局面要怎么偏？

#### 8. 章末方法论（chapter_ending）

**核心定义：** 章末目标不是"收束完整"，而是"留下后劲"。

**四种有效落点：**
- **余韵型**：前面累积的情绪在最后一拍落下去
- **反打一刀型**：最后一句/动作改写前文局势理解
- **判决型**：主角或他人做出直接推动下一章的决定
- **轻喜回钩型**：靠一句回钩/歪刀/关系偏移收

**章末五问：**
1. 是不是停在"变化"之后？
2. 有没有把这一章最值钱的东西收住？（情绪/新信息/新决定/新危险）
3. 有没有把下一章的门打开？
4. 是不是停在解释上？（解释最容易把张力解释没）
5. 删掉最后三句，损失大不大？（没差说明章末没起作用）

**章末最小计划：**
- 这一章最值钱的变化是什么？
- 准备收在情绪/动作/信息/决定上？
- 下一章第一拍要接什么？
- 这一收是柔切/硬切/反打/余韵？

#### 9. 去AI味方法论（anti_ai_voice）

**核心定义：** AI味不是词不够华丽，而是：事情没真正发生只被总结、情绪没长在动作里只被解释、人物没有自己的嘴只替作者说明、句子太平均像模板复写。

**底层四原则：**
1. **用动作承载情绪**：把"她很难过"改成可被看见的行为/停顿/距离/物件变化
2. **用局面承载信息**：信息出现后，人物怎么改口/谁占上风/下一步怎么变
3. **用身份承载说话方式**：同一句话不同身份有不同版本
4. **用节奏打破平均**：有的句子出拳、有的停顿、有的补关键细节、有的短到只剩一个动作

**五类AI味典型表现：**
- **空泛总结型**：心中五味杂陈/内心翻涌/感慨万千 → 删总结，写身体反应和场上物件
- **套话氛围型**：空气凝固/时间静止/气氛微妙 → 写谁停了/谁没接话/谁退了一步/哪个小声音变刺耳
- **形容词堆叠型**：清冷/矜贵/禁欲/疏离 → 删一半标签，补一个制造压迫感的动作
- **解释腔对白型**：边吵边讲设定/情绪说明白/像讲道理 → 回对白模块修
- **平均工整型**：句子一样长/一样完整/都在解释 → 短句插刀/动作截断/话没说完/解释挪出台词

**常用改写法：**
- 总结句改动作句（谁碰了什么/谁没碰什么/谁拉远拉近/谁该说没说）
- 情绪句改反应链（看到什么→身体卡哪一下→做什么掩饰）
- 气氛句改场内变化（谁安静/哪个声音被听见/哪个动作压住局面）
- 标签句改判定动作（做一个让别人收声/退步/改口的动作）
- 说明书对白改压迫对白（立场/条件/后果/最后通牒）

**模块配合顺序：** 先修结构硬伤 → 再修人物关系硬伤 → 最后压掉空泛解释模板感

#### 10. 一致性审查方法论（consistency_review）

**强制执行原则：** 每一章写完必须过审查，不是有问题再看，而是写完必看。

**六种必查一致性：**
1. **剧情逻辑一致性**：关键事件有没有前提？关键决定有没有触发？关键变化有没有后果？
2. **人物目标一致性**：主角这章最想做什么？目标有没有无故漂移？
3. **情绪与关系一致性**：情绪有没有来路？关系温度前后对不对得上？
4. **身体与信息状态一致性**：伤势/疲劳/秘密/误会/已知信息有没有丢？
5. **场景与转场一致性**：读者会不会迷路？上一场余力有没有带到下一场？
6. **章末承接一致性**：有没有收住本章最值钱变化？下一章第一拍能不能自然接？

**补充审查清单：**
- 章内结构（开头抓手/中段推进/局面变化/章末停点）
- 开头与前三章（300字抓手/避免天气日常背景/主角尽快出场/卖点危机显形）
- 本章推进（核心事件/局面变化/不是水章/至少推进一项）
- 信息投放（避免大段背景直灌/信息跟着冲突/不要一口气塞太多设定）
- 场景有效性（每场戏有目标/阻碍/变化/可删段落判断）
- 语言表达（避免空泛抒情/避免同义重复/句子顺滑/对白符合身份/情绪落地）
- 连载连续性（不忘前文承诺/不突然引入过多新设定/伏笔推进/服务故事引擎）

**问题回跳机制：** 转场→transition、对白→dialogue、章末→chapter_ending、因果→plot_logic、人设→character_consistency

### 四、逐章写作硬规则

**每章必须回答四件事：**
1. 主角这章想做什么？

### 八、核心设计哲学总结

1. **问题分层**：不把所有写作知识堆在一个文件里，按问题层级拆分模块
2. **先诊断后开药**：先判断问题在哪个环节，再调用对应模块，不做泛建议
3. **底层优先**：先修因果和人物（底层），再修场景和章节（中层），最后修文风（表层）
4. **先计划后落笔**：每个环节动笔前先填"最小计划卡"，写不清就不写正文
5. **素材驱动**：先匹配本地语料和正例，总结结构，再写新内容，不靠空想
6. **强制质检**：每章写完必须过一致性审查，不能直接交稿
7. **最小骨架思维**：每个阶段都先压缩成最小可验证骨架，再展开
8. **变化驱动叙事**：每场戏、每章都必须有变化（局面/关系/信息/情绪），没变化就删并压缩
9. **成交优先**：网文开头是成交区不是铺垫区，前几百字就要抓住人
10. **可见原则**：能用动作不用总结，能用对白不用解释，能写具体不写抽象

### 八、01_小说创作杂项核心文件总结

#### 1. AI学习指南（杂项设定/学习指南.md）
**针对性解决问题：** 批评"纯文本描述没有互动与剧情起伏"，这不是字数问题是写法问题
**阅读四问法：** ①开篇怎么写的第一段做什么让读者想继续？②人物怎么活的对话互动情绪变化不是工具人？③色情怎么嵌入叙事的不是打完再搞而是融入情节本身？④章节怎么收结尾留什么让人想看下一章？
**逐章拆解6步模板：**
- ①开篇（前3段）：记录开篇方式/第一句/钩子/色情出现时机
- ②对话与互动：计数对话段数/角色语气区分度/一来一回张力/是否通过对话交代信息
- ③场景与氛围：五感覆盖/环境与人物情绪绑定/摘3句好句
- ④战斗与色情融合⭐（重点中的重点）：打了再搞还是边打边色/色情通过什么方式融入战斗/战斗结果胜负是否和色情有关、明确反例对比（错误写法：白夜战斗→打败敌人→救朝日→朝日被榨精→获救；正确写法：白夜战斗中被攻击裆部→边战斗边压制勃起→敌人利用分心发动攻击→战斗结果受色情影响）
- ⑤人物心理与成长：内心独白/情绪变化/心理弱点（不只是物理弱点）
- ⑥章节结构与章末：起承转合/章末钩子/情绪曲线
**学习技法分类清单：** 开篇手法（梦境/电话/日常/行动/对话开场法）、战斗色情融合手法（装备压迫/针对性攻击/环境利用/姿势暴露）、对话制造法（多角色同时对话/三种语气搭配/通过对话交代信息）、章末钩子类型（敌人突然说话/发现自己暴露危险中/内心怀疑焦虑/画面定格危机即将发生瞬间）

### 九、整体写作风格与创作哲学总结

#### 1. 写作风格核心特征
- **直白专业不客套**：助手身份是"写文搭子"，不用客服腔，写得不好直接指出来，少用表情符号，自称"我"称呼"你/老板"不用"您"
- **细节极致导向**：榨精手法必须写具体步骤，不能省略过程；紧身衣必须"描边"不能只写"穿着紧身衣"；五感至少覆盖3种必须有触觉视觉
- **动作-感受强绑定**：每写一个动作必须紧接着写身体反应和主观感受，禁止旁观者式纯动作记录
- **心理抵抗是核心张力**：即使身体爽到失控，内心必须保留羞耻/愤怒/不甘的抵抗，"清醒沉沦"是最核心戏剧张力
- **反差萌是人物灵魂**：外冷内热/嘴硬心软/强大又脆弱/表面正经内心好色，每个角色必须有表里反差
- **节奏变化必须有**：榨精不能一个速度撸到底，必须有蓄精→寸止→冲刺→突破→追击五段完整节奏，轻重交替快慢切换突然停住
- **日常化堕落最高境界**：色情不是特殊事件而是融入日常，最色的不是强行榨精是整个社会/群体都觉得这是正常的理所当然

#### 2. 创作方法论核心
- **分层查阅优先级**：SOUL核心原则→STYLE_CODES选节奏文风视角→对应题材NOTES笔记→MEMORY回顾前文人物状态→最近3章已写内容→人物卡确认性格→需要细节再查原始参考文件
- **先大纲后正文**：写100-200字大纲确认后再写正文，不闷头硬写
- **写完必自查28项**：红线禁令→核心原则→角色叙事→技术细节，按优先级逐一检查
- **每章必须有变化**：每章必须推进至少一个剧情点（新敌人/新情报/关系变化/能力觉醒/被改造），不能纯色情注水
- **章末必须留钩子**：悬念/敌人后手/新危机/阴谋发现/身体异常变化/"他还不知道这只是开始"
- **情绪曲线必须有起伏**：平静→紧张→快感→危机→爆发→余韵→新悬念，不能一直平
- **机制先于情感**：先铺设定机制（精能体系/淫纹效果/道具功能），情感是机制副产品，不要先写"爱上了"要先写"身体已经被改造得只能高潮了"
- **素材驱动写作**：优先查整理好的NOTES笔记，需要细节再回原始素材，不靠空想靠积累的技法库词库场景库
- **批量子代理工作流**：大量素材阅读和草稿生成通过sub-agent派发，用标准化9段模板保证输出质量，主agent做最终整合和正文创作

```
合并到正式深度阅读笔记
```

**优势**：批次内并发度最高；草稿可独立审查、出错可重跑单本；主代理合并时按段号排序，避免写冲突。

#### 子代理标准执行流程（强制顺序，不可跳过）
```
步骤1：ls 源目录 → 确认文件存在（强制第一步）
  ↓
步骤2：打开本SA的草稿模板文件（已在子类型文件夹中预生成）
  ↓
步骤3：分批读取源小说（每批3-5章 或 单部作品）
  ↓
步骤4：读完一批立即Write/Edit写入草稿对应§4.X段（不积压上下文）
  ↓
步骤5：重复3-4直到全部读完
  ↓
步骤6：末尾填写5行完成报告
  ↓
步骤7：向主代理回报1行摘要
```

#### 分批粒度规则（防止上下文超限）
| 场景 | 分批规则 | 写入时机 |
|---|---|---|
| 单部独立作品 | 整部 = 1批 | 读完立即写 |
| 多部聚合包 | **1部 = 1批** | 每读完1部立即追加写入，N部=N套9段内容 |
| 超长作品(单部>300KB) | 每3-5章 = 1批 | 每读完一批立即追加 |
| 多文件系列子段 | 按§0清单分配的文件逐批读取 | 每读完一批文件立即追加 |

#### 多作品SA拆分规则（v1.1）
- 负责**1部作品**的SA：**不拆分**，保持原样
- 负责**2部作品**的SA：**拆分为2个独立子任务**（后缀-1、-2），每个1部作品
- 负责**3部作品**的SA：**拆分为3个独立子任务**（后缀-1、-2、-3）
- 负责**4部及以上作品**的SA：**不拆分**（保持单个SA负责多部）
- **系列/合集作品**（文件名含"系列"、"合集"、"乱七八糟"等）：算1部，不拆分

**子任务ID命名规则**：原SA-ID基础上追加作品序号后缀（如SA-02-A→SA-02-A-1、SA-02-A-2）
**草稿文件命名统一**：`{SA-ID}_{作品名}_草稿.md`（不管是否拆分，必须带作品名）

**拆分目的**：减小单个SA上下文压力，提高并发度，避免一个SA处理多部作品导致上下文超限。

### 四、子代理任务的标准格式（v3.2 9段结构）

> 多部聚合包：N部作品=N套完整9段，按顺序追加到同一个草稿文件。段号固定，顺序不可变，缺一不可。

| 段号 | 段名 | 硬性要求 | 核心内容 |
|:---:|---|:---:|---|
| **§1** | **作品定位** | — | 类型/流派/情绪基调/目标读者画像/核心卖点 |
| **§2** | **关键章节** | 表格 | 列：章节号/标题/剧情浓度(1-5)/核心看点；选3-10个关键章节 |
| **§3** | **完整套路** | 5-12段 | 核心叙事模式+反转节点+情绪曲线+节奏推进公式 |
| **§4** | **完整情节** | — | 梳理完整剧情脉络，分阶段概括（~1000字） |
| **§6** | **关键R18情节元素总结** | **8-20条，每条≥500字（硬要求）** | 总结原文关键R18段落，保留动作/对白/心理/核心细节（姿势/部位/节奏/主导权）；按题材侧重特色元素 |
| **§7** | **内容标签判定** | **只用R1/R2/R3，命中/不命中** | 见雷点判定v3严格版 |
| **§8** | **写作技法** | **3-8项，每条附100字以内原文示例** | 技法名称+使用章节+效果分析+原文示例（≤100字）+可复用建议 |
| **§9** | **SKILL升级建议** | **≥2条（硬要求）** | 具体可操作建议，对应references可补充的内容 |

**注意**：§5「重复模式/模板化桥段」已在v3.2删除，相关内容融入§3完整套路即可。段号从§1直接跳到§6为历史遗留编号，保持不变。

### 五、质量检查标准

#### 1. §6关键R18情节元素检查（最核心）
- **条数检查**：8-20条（按R18浓度决定，不得少于8条）
- **字数检查**：每条≥500字（硬性要求，不足必须补充）
- **内容检查**：必须保留核心动作（姿势/部位/节奏）、角色对白、心理描写、主导权关系、关键细节（感官/环境/道具）
- **题材侧重检查**：必须覆盖对应题材的特色元素（如战队特摄必须有紧身衣/胯下攻击/战败羞辱）
- **验证命令**：`python verify_r18_length.py 草稿/pixiv小说_{XX}_*.md --min 500 --max 20`

#### 2. §7雷点判定检查
- **标签完整性**：必须出现R1/R2/R3三个标签各1次
- **措辞正确性**：每个标签只能判定为「命中」或「不命中」
- **严禁措辞检查**：不得出现「边缘命中/轻度命中/擦边/子标签」等模糊词
- **默认值规则**：R2/R3默认不命中（不描写R18G血腥/父子乱伦）
- **验证命令**：
  ```bash
  grep -E "R[123].*(命中|不命中)" 草稿/*.md  # 必须有匹配
  grep -nE "边缘命中|轻度命中|擦边|子标签" 草稿/*.md  # 必须无匹配
  ```

#### 3. §8写作技法检查
- **条数检查**：3-8项
- **原文示例检查**：每条必须附100字以内原文示例
- **维度覆盖**：建议覆盖叙事装置/节奏控制/对话风格/视角选择/设定融合/伏笔回收/情绪调度等维度

#### 4. §9升级建议检查
- **条数检查**：≥2条（硬要求）
- **质量要求**：具体可操作，对应references可补充内容，不能空泛

#### 5. 结构完整性检查
- **9段齐全**：§1-§4、§6-§9共8段（§5已删除）缺一不可
- **段号顺序**：必须按固定顺序，不可乱序
- **完成报告**：草稿末尾必须有标准5行完成报告

#### 6. detailed_audit.json审计指标
从审计JSON可以看到质量量化指标：
| 字段 | 含义 | 合格标准 |
|---|---|---|
| `s6_count` | §6 R18元素条数 | 8-20条 |
| `avg_s6_len` | §6每条平均字数 | ≥500字 |
| `total_s6_chars` | §6总字符数 | 按条数×500计算 |
| `has_s7` | 是否有§7雷点判定 | true |
| `has_s8` | 是否有§8写作技法 | true |
| `has_s9` | 是否有§9升级建议 | true |
| `has_report` | 是否有完成报告 | true |
| `is_v32_completed` | 是否符合v3.2格式 | true |
| `sections` | 实际包含的段落列表 | 必须包含标准9段 |

6. ❌ §6条数少于8条

8. ❌ 使用"边缘命中/擦边/轻度命中"等模糊雷点判定

### 六、批量处理系统演进总结

从v1→v2→v2.3→v3.2的演进路径体现了以下设计原则：

1. **上下文管理优先**：从"读完再写"→"分批读取立即写入"，解决子代理上下文超限问题
2. **硬要求简化强化**：从复杂多维度要求→单一核心硬要求（§6字数），其余自由发挥，降低子代理认知负担同时保证质量底线
3. **雷点判定严格化**：从模糊标签→严格三标签二元判定，避免歧义
4. **题材差异化**：统一框架下保留每个题材的特色R18元素池和重点关注点
5. **并发度最大化**：同批次完全并行+多作品SA拆分，最大化利用子代理并发能力
6. **质量可验证**：提供自动化验证脚本和量化审计指标（detailed_audit.json）
7. **规范自包含**：v3.2起每个输入输出规范文件自包含所有必要信息（路径+流程+格式+硬要求+禁止清单），子代理读这一个文件即可开始工作
8. **原文沉淀机制**：v3.2新增"§8每条附100字以内原文示例"要求，直接沉淀可复用原文片段到知识库

这是一套经过实战验证的大规模R18小说素材批量处理系统，通过标准化流程+并行化执行+严格质量检查，实现了高效、高质量的小说深度阅读笔记批量生产。
