# 星辰模块

这是一个专门处理二次元萌系奇幻纯爱文风的独立模块，核心范本为冷酷杀手攻×软萌正太受的《星辰之下》。

这次模块的组织方式，已经按 `sources/raw/` 里的原文资料重排，不再把萌系文风仅仅当成"用几个叠词"，而是拆成六层：

- 新手能立刻上手的萌系词库与基础语气
- 强反差人设的三种建立方式
- 日常萌点的10+种设计模板
- 12个情感阶段的递进节奏
- 核心意象（星辰/狼羊/约定/挂坠）的系统运用
- 纯爱风格亲密场景的节奏把控与停止点设计

## 模块目标

这个模块主要解决：

- 萌不起来，只会生硬堆砌叠词
- 反差人设只写设定，没有具体行为体现
- 日常互动像流水账，没有萌点
- 情感推进太快或太慢，节奏失控
- 攻方太冷酷像变态，没有温柔内核
- 受方太刻意卖萌，显得假
- 意象用一次就丢，没有贯穿感
- 亲密场景要么太露骨破坏萌感，要么太清水没有张力
- 心理描写单薄，只有"心跳加速"没有层次
- 傲娇吐槽写得像真嫌弃，没有口是心非的甜感

## 星辰风的适用题材

星辰风最适合以下题材：
- 正太纯爱（年龄差攻受，12岁攻×8岁受为范本）
- 杀手/非人/超能力者 × 普通人类小孩
- 强反差人设CP（外冷内热攻 × 软萌活泼受）
- 治愈系日常甜文
- 带有轻奇幻/超能力元素的校园/家庭文
- 守护系纯爱（攻默默守护受）
- 带有"狼与羊"隐喻的跨种族/跨身份恋爱

不适合：
- 成人向现实主义题材
- 强剧情悬疑向（萌系文风会削弱紧张感）
- 虐文（虽然有虐点但核心是甜）
- 女性向BG（需要调整萌点分配）

## 现在这套模块怎么读

推荐按这个顺序用：

1. 先读 [source_index.md](source_index.md)，知道原文各章节重点学什么。
2. 再读 [tutorial.md](tutorial.md)，建立统一的星辰风写作框架。
3. 写作或改稿时，按 [runtime.md](runtime.md) 的顺序诊断问题。
4. 去 [good_examples.md](good_examples.md) 找2-4个近似正例。
5. 去 [bad_examples.md](bad_examples.md) 找1-2个近似反例。
6. 先定人设反差，再写正文。

## 星辰风的核心识别特征

如果你写出来的文字有以下特征，说明你抓到了星辰风的精髓：

1. **强反差人设落地**：攻方前一秒还在冷酷杀人/打架，后一秒就给受方刷碗/弹脑瓜/脸红
2. **萌点有层次**：不是只有"好可爱"，而是外貌萌+语言萌+动作萌+心理萌+小脾气五层叠加
3. **意象贯穿**："星辰"不只是名字，还代表温暖、光明、守护；"狼与羊"不只是比喻，还伴随着身份焦虑
4. **口是心非的傲娇**：攻方嘴上说"恶心""别过来"，身体却很诚实地靠近/保护/温柔
5. **纯爱克制**：亲密场景点到为止，在最关键的时刻停下来，用责任和爱战胜欲望
6. **童趣日常**：玩扑克、爬楼梯、洗澡、写作业、上学放学——普通日常写出甜味
7. **眼泪有重量**：受方一哭攻方立刻投降，眼泪是终极武器但不能滥用
8. **占有欲双向**：不只是攻宠受，受也会吃醋、宣示主权、咬胳膊惩罚

## 说明

- 当前模块的"主参考"是 `sources/raw/星辰之下.txt`，所有技法均从原文提取，不凭空编造。
- 正例强调可迁移的萌点结构，不鼓励照搬原文句子皮相。
- 反例不是否定某类写法，而是指出在星辰风语境下会破坏萌感的常见错误。
- 亲密场景部分总结叙事节奏、心理描写、语言风格的技法，保持纯爱风格，不刻意露骨但也不回避原文内容。
- 全文统一使用"星辰"，绝对不出现"星原子"错别字。
